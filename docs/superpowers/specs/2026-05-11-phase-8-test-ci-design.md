# Phase 8 — CI Test Workflow Design

**Date:** 2026-05-11
**Status:** Approved (brainstorm)
**Scope:** Add GitHub Actions workflow that runs the full quality gate on PR + push to main. Zero secret setup. Solo/private repo profile.

## Goal

Catch regressions in `pnpm test`, `pnpm build:typecheck`, and `pnpm build` before they reach `main`. Provide a green/red signal on every PR and every direct push.

## Out of Scope

- Branch protection rules (user-driven setting, can be enabled after the workflow lands).
- Real Supabase secrets in CI (deferred until the repo accepts external PRs).
- Linux/macOS runners.
- Concurrency cancellation, matrix builds, nightly schedules.
- Test-result artifacts, coverage upload, or status badges.

## Trigger

```yaml
on:
  pull_request:
  push:
    branches: [main]
```

`pull_request` fires on opened/synchronize/reopened by default — enough to catch every PR update.
`push: branches: [main]` catches direct-to-main merges and post-merge state.

## Runner

`windows-latest`. Matches local dev environment and `release.yml`. Avoids cross-OS surprises around the better-sqlite3 ABI swap whose Windows-specific paths (`shell: process.platform === 'win32'`) are not exercised on Linux.

## Steps

1. `actions/checkout@v4`
2. `pnpm/action-setup@v4` with `version: 10`
3. `actions/setup-node@v4` with `node-version: '20'` and `cache: 'pnpm'`
4. **Write stub .env** via shell heredoc:
   ```
   MAIN_VITE_SUPABASE_URL=https://stub.supabase.co
   MAIN_VITE_SUPABASE_ANON_KEY=stub-anon-key
   RENDERER_VITE_SUPABASE_URL=https://stub.supabase.co
   RENDERER_VITE_SUPABASE_ANON_KEY=stub-anon-key
   ```
5. `pnpm install --frozen-lockfile` — triggers `postinstall` → `scripts/rebuild-sqlite.mjs --runtime=electron` → marker = `electron-v130`.
6. `pnpm test` — `pretest` flips to node ABI, vitest runs the full suite, `posttest` restores electron ABI.
7. `pnpm build:typecheck` — `tsc -p tsconfig.web.json --noEmit && tsc -p tsconfig.node.json --noEmit`.
8. `pnpm build` — `electron-vite build` (bundles renderer + main + preload).

## Workflow File

`.github/workflows/test.yml`:

```yaml
name: Test

on:
  pull_request:
  push:
    branches: [main]

jobs:
  test:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'

      - name: Write stub .env
        shell: bash
        run: |
          cat > .env <<'EOF'
          MAIN_VITE_SUPABASE_URL=https://stub.supabase.co
          MAIN_VITE_SUPABASE_ANON_KEY=stub-anon-key
          RENDERER_VITE_SUPABASE_URL=https://stub.supabase.co
          RENDERER_VITE_SUPABASE_ANON_KEY=stub-anon-key
          EOF

      - name: Install
        run: pnpm install --frozen-lockfile

      - name: Test
        run: pnpm test

      - name: Typecheck
        run: pnpm build:typecheck

      - name: Build
        run: pnpm build
```

## Why Each Choice

- **Windows-latest only:** matches release.yml; ABI swap paths exercised on the OS that actually exercises them.
- **pnpm 10:** matches `pnpm-lock.yaml` (generated by pnpm v10.30.1).
- **Node 20:** matches release.yml.
- **`cache: 'pnpm'`:** keyed on `pnpm-lock.yaml` hash. Reduces install time on cache hit.
- **Stub .env:** zero secret-management overhead; `release.yml` validates real-env build path before shipping, so CI's job is regression detection, not duplicate validation.
- **Quality gate order (test → typecheck → build):** test ordering matches local; typecheck before build means a TypeScript regression fails the cheaper step first.
- **No `concurrency:` block:** repo is low-volume; in-flight runs acceptable. Can add later if needed.

## Acceptance

1. Workflow file lands on main.
2. CI runs once on push-to-main, finishes green within ~6 min (estimate: 1 min install, ~30 s rebuild, ~30 s test, ~10 s typecheck, ~30 s build, plus runner startup).
3. Open a no-op PR → CI runs on pull_request event → green.
4. Intentionally break a test in a PR (revert before merge) → CI red.
5. Marker file (`node_modules/.better-sqlite3-abi`) ends as `electron-v130` after the workflow (verified via posttest run before build).

## Risk

- First run cache miss: full source build of better-sqlite3 = ~30–60 s additional. Subsequent runs hit cache.
- pnpm 10 + `pnpm/action-setup@v4` compatibility: confirmed by `release.yml` already using the same combination.
- Windows runner cost = 2× Linux minutes. For solo repo + few PRs, negligible.
- Stub .env causes Vite to embed `https://stub.supabase.co` in the bundle. Bundle never runs at this stage (no `pnpm package`), so no risk of stub URLs leaking into a release artifact.

## Follow-ups (deferred)

- Branch protection rules requiring `test` workflow success.
- Concurrency cancellation for redundant in-flight PR runs.
- Status badge in README.
- Test result / coverage artifact upload.
- Linux/macOS matrix once non-Windows users exist.
