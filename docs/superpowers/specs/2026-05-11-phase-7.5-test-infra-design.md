# Phase 7.5 — Test Infra ABI Swap Design

**Date:** 2026-05-11
**Status:** Approved (brainstorm)
**Scope:** Local-only test infra fix. No CI changes. Prerequisite for phase 8.

## Goal

Make `pnpm test` green on Windows. Restore confidence in vitest before adding new feature code in phase 8.

## Problem

- `postinstall` runs `electron-rebuild -v 41.5.0 --only better-sqlite3 --force`. This compiles the better-sqlite3 native binding for Electron 41's bundled Node (Node 22, `NODE_MODULE_VERSION=130`).
- `pnpm test` runs vitest under host Node (devDep `@types/node@^25` indicates Node 24/25, `NODE_MODULE_VERSION=137`).
- ABI mismatch causes `require('better-sqlite3')` to fail when `bindings()` cannot load the prebuilt `.node` file.
- Symptom: 21 vitest failures across 6 files (`tests/main/{audit-end-to-end,chat-service,db,projects-repo,scanner-end-to-end,workspace-service}.test.ts`). All trace to the same `new Database(...)` call in `src/main/db/client.ts:14`.
- Typecheck (`pnpm build:typecheck`) and build (`pnpm build`) are green. The shipped product is not affected.

## Solution Summary

ABI-swap scripts around test runs. Each command (`pnpm test`, `pnpm test:watch`, `pnpm dev`) leaves the repo in a known-good state:

```
pretest        → rebuild better-sqlite3 for host node ABI
test           → vitest run
posttest       → rebuild better-sqlite3 for electron ABI

pretest:watch  → rebuild for node ABI
test:watch     → vitest watch
posttest:watch → rebuild for electron ABI

predev         → rebuild for electron ABI (idempotent — no-op when already correct)
```

Marker file in `node_modules/` records the current ABI; rebuild script short-circuits when target matches. Steady-state cost = one rebuild per direction switch (~15 s), not per command.

## Components

### 1. `scripts/rebuild-sqlite.mjs`

Single Node ESM script. Accepts `--runtime=node|electron`. Responsibilities:

- Read marker file `node_modules/.better-sqlite3-abi`.
- If marker matches requested runtime → exit 0 (no rebuild).
- Otherwise:
  - `--runtime=electron`: shell out to `electron-rebuild -v 41.5.0 --only better-sqlite3 --force` (current postinstall behavior).
  - `--runtime=node`: invoke `pnpm rebuild better-sqlite3 --build-from-source` with `npm_config_runtime=node` and `npm_config_target=<current host node version>`. (Source build, not prebuilt download — prebuilt may not exist for the host node major.)
- Write marker file with new runtime tag.
- Print one-line status: `rebuilt better-sqlite3 for node v25.x.x` or `skipped (already electron-v130)`.

Exit non-zero on rebuild failure. Do not swallow.

### 2. `package.json` script changes

Add/modify:

```json
{
  "scripts": {
    "predev": "node scripts/rebuild-sqlite.mjs --runtime=electron",
    "dev": "electron-vite dev",
    "pretest": "node scripts/rebuild-sqlite.mjs --runtime=node",
    "test": "vitest run",
    "posttest": "node scripts/rebuild-sqlite.mjs --runtime=electron",
    "pretest:watch": "node scripts/rebuild-sqlite.mjs --runtime=node",
    "test:watch": "vitest",
    "posttest:watch": "node scripts/rebuild-sqlite.mjs --runtime=electron",
    "prepackage": "node scripts/rebuild-sqlite.mjs --runtime=electron",
    "package": "pnpm build && electron-builder --win",
    "postinstall": "node scripts/rebuild-sqlite.mjs --runtime=electron"
  }
}
```

Existing `postinstall` line is replaced (was `electron-rebuild -v 41.5.0 --only better-sqlite3 --force`). New script wraps it with the marker check.

### 3. ABI marker file

Path: `node_modules/.better-sqlite3-abi`
Contents: single line, one of `node-v<NODE_MODULE_VERSION>` or `electron-v130`.
Lifecycle: written by rebuild script. Removed implicitly when `node_modules` is wiped.
Gitignore: lives under `node_modules`, already ignored.

### 4. CTRL-C safety net

`test:watch` quit via Ctrl-C does not always run `posttest:watch`. The `predev` script provides recovery: next `pnpm dev` detects marker = `node`, rebuilds to `electron`, marker updates. Idempotent and cheap when already correct.

Same `predev` guard applies to `prepackage` so a `pnpm package` after a watch session that exited mid-rebuild is also safe.

## Edge Cases

- **Fresh clone:** `pnpm install` triggers `postinstall` → marker = `electron`. First `pnpm test` swaps to `node` → first test cycle pays ~30 s round-trip. Subsequent cycles only pay one direction's rebuild when needed.
- **CI (`.github/workflows/release.yml`):** runs `pnpm install` + `pnpm package`. Both flow through electron rebuild via `postinstall` and `prepackage`. No test step in CI today, so no swap occurs. Unaffected.
- **Other native deps:** `node-pty` is in `pnpm.onlyBuiltDependencies` but not referenced from any test file (grep confirmed). Electron-only path. Not rebuilt by this script.
- **Source-build fallback:** if `pnpm rebuild ... --build-from-source` cannot compile (missing build tools), script must print clear remediation (`Install Visual Studio Build Tools` on Windows, `xcode-select --install` on macOS, `build-essential` on Linux) and exit non-zero. Do not fall back to a prebuilt that may mismatch ABI.
- **Node version drift:** if a contributor upgrades host Node, the marker still says `node-vNNN` for the old version. To be safe, marker should include `NODE_MODULE_VERSION` and rebuild when it differs from `process.versions.modules`, not just runtime label.

## Acceptance

- `pnpm install` clean → marker = `electron-v130`.
- `pnpm test` → 146/146 pass; marker = `electron-v130` after posttest.
- `pnpm test:watch` → tests run; Ctrl-C; `pnpm dev` succeeds (predev restores).
- `pnpm dev` → predev no-op (already electron); app starts.
- `pnpm package` → prepackage no-op (already electron); installer builds.
- Build + typecheck remain green.

## Out of Scope

- CI test workflow (deferred; phase 8 may add).
- Switching better-sqlite3 mock strategy (rejected approach B in brainstorm).
- Rebuilding node-pty for tests (not consumed by tests).
- Cross-runtime symlink/two-binary cache (rejected approach C in brainstorm).

## Risk

- Rebuild from source requires platform build tools on every dev machine. Already required by current `electron-rebuild` postinstall — no new burden.
- ~15 s added latency per ABI direction-switch. Acceptable for stateless invariant.
- `pnpm rebuild` semantics across pnpm versions: validate on the pnpm version in `pnpm-workspace.yaml` / CI before locking script.
